Attribute rules

Definition of Attribute

In general, an attribute is a characteristic. In HTML, an attribute is a characteristic of a page element, such as font size or colour. Attributes are used to amplify a tag.Attributes are used with elements and they must always have quotes . Attributes are having a name and a value. There must not be two or more attributes on the same start tag whose names are a case-insensitive match for each other.

Rules:

  • The value must be unique amongst all the IDs in the element's home sub tree and must contain at least one character. The value must not contain any space characters.
  • Attribute value must have qoutes()
  • Attribute name needs to be lowercase
  • Element name have private attributes
  • Public element can go on any element
  • No double quotes inside double quotes
  • Class value can have more than one separated by space and not contains period
  • You can not use form inside form
  • You can not put html in summary
  • You can not put html in button
  • You can not put <sup> inside <input>
  • If the value of an attribute using the unquoted attribute syntax is followed by a "/" character, then there must be at least one space character after the value and before the "/" character.
  • The <src> attribute must be present, and must contain a valid non-empty URL potentially surrounded by spaces referencing a non-interactive, optionally animated, image resource that is neither paged nor scripted
